# Руководство по созданию иконок для NAS пакетов (x86_64)

## Обзор

Этот документ описывает требования к иконкам для всех типов NAS пакетов на архитектуре x86_64.

## Структура директорий

```
packages/
├── synology/
│   └── icons/

│       ├── icon_120.png

├── qnap/
│   └── icons/
│       └── icon.png
├── asustor/
│   └── icons/
│       └── icon.png
└── truenas/
    └── icons/
        └── icon.png (опционально, для Kubernetes)
```

## Требования по платформам

### Synology DSM

**Формат:** PNG
**Размеры:**
- `icon_72.png` - 72x72 пикселей
- `icon_120.png` - 120x120 пикселей
- `icon_256.png` - 256x256 пикселей

**Особенности:**
- Поддержка прозрачности (альфа-канал)
- Рекомендуется использовать светлый фон или прозрачный
- Иконки должны хорошо выглядеть на темном фоне Package Center

### QNAP QTS

**Формат:** PNG
**Размер:** `icon.png` - 128x128 пикселей (рекомендуется) или 64x64

**Особенности:**
- Поддержка прозрачности
- Квадратный формат обязателен

### Asustor ADM

**Формат:** PNG
**Размер:** `icon.png` - 128x128 пикселей (рекомендуется) или 64x64

**Особенности:**
- Поддержка прозрачности
- Квадратный формат обязателен

### TrueNAS SCALE

**Формат:** PNG (опционально, для Kubernetes)
**Размер:** `icon.png` - 128x128 пикселей

**Особенности:**
- Используется в Kubernetes UI (если поддерживается)
- Поддержка прозрачности

## Процесс создания

### Шаг 1: Дизайн

1. Создайте векторную иконку в формате SVG или AI
2. Используйте простой, узнаваемый дизайн
3. Убедитесь, что иконка читаема в маленьком размере

### Шаг 2: Экспорт

1. Экспортируйте иконку в PNG для каждого требуемого размера
2. Используйте высокое качество (без сжатия) для исходных файлов

### Шаг 3: Оптимизация

1. Оптимизируйте PNG файлы с помощью инструментов:
   - `pngquant` - уменьшение размера файла
   - `optipng` - оптимизация без потери качества
   - `ImageMagick` - конвертация и оптимизация

### Шаг 4: Тестирование

1. Установите пакет на тестовое устройство
2. Проверьте отображение иконки в Package Center/App Center
3. Убедитесь, что иконка хорошо выглядит на разных фонах

## Инструменты

### Графические редакторы

- **GIMP** (бесплатный) - https://www.gimp.org/
- **Inkscape** (бесплатный, векторный) - https://inkscape.org/
- **Adobe Illustrator** (платный) - https://www.adobe.com/products/illustrator.html

### Оптимизация

- **pngquant** - https://pngquant.org/
- **optipng** - http://optipng.sourceforge.net/
- **ImageMagick** - https://imagemagick.org/

### Пример команды для создания иконок из SVG

```bash
# Создание иконок для Synology из SVG
inkscape icon.svg --export-filename=icon_72.png --export-width=72 --export-height=72
inkscape icon.svg --export-filename=icon_120.png --export-width=120 --export-height=120
inkscape icon.svg --export-filename=icon_256.png --export-width=256 --export-height=256

# Оптимизация
pngquant --quality=65-80 icon_72.png icon_120.png icon_256.png
```

## Временные иконки

До создания финальных иконок можно использовать временные placeholder'ы:

1. Создайте простую иконку с текстом "IP-CSS"
2. Используйте базовые цвета проекта
3. Замените на финальные иконки перед релизом

## Рекомендации

1. **Простота** - иконка должна быть простой и узнаваемой
2. **Контраст** - используйте достаточный контраст для читаемости
3. **Масштабируемость** - иконка должна хорошо выглядеть в разных размерах
4. **Брендинг** - используйте цвета и стиль проекта IP-CSS
5. **Уникальность** - иконка должна отличаться от других пакетов
